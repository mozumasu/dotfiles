# 事実ベース回答の原則

## 根拠の明示

- 確認していないことを断定しない。回答の根拠を必ず示す:
  - コード: ファイルパスと行番号（例: src/store/counter.ts:12）
  - ドキュメント: 参照した URL
  - 動作: 実際に実行したコマンドとその出力
- 根拠を確認できなかった場合は「確認できなかった」と明示し、
  推測で補う場合は推測であることを先に断る。
- 「わからない」と答えることは失敗ではない。誤った断定が失敗である。

## 検証してから完了報告

- コード変更後は lint / test / 型チェックを実行し、その出力を確認してから
  完了と報告する。未実行のまま「動くはず」と報告しない。
- テストが失敗した場合は失敗した事実と出力をそのまま報告する。

## コードの動作はコードで確認

- 「〜という実装になっているはず」ではなく、Grep / Read で実コードを
  確認してから述べる。コメントではなく実装から動作を判断する。

# コメントとコミットメッセージの書き分け

情報の置き場所を次で使い分けること:

- **コードコメント**: コードから読み取れない制約・不変条件のみ (例: 「この API は分/時フィールドを無視する」)。
  原則 1〜3 行以内。以下はコメントに書かない:
  - 変更の経緯・理由 (「〜のために追加」「〜だったので修正」「〜を防ぐため」) → コミットメッセージへ
  - レビュアー向けの説明 (「この変更で〜が直る」「従来は〜だった」) → PR 本文へ
  - 次の行が何をするかの言い換え
  - Issue/PR 番号への参照 (git blame で辿れる)
- **コミットメッセージ**: 変更の why (なぜこの変更が必要だったか、どんな問題があったか、なぜこの方法を選んだか)
- **PR 本文**: レビュアー向けの背景・設計判断・動作確認結果・スクリーンショット
